# Soul

Build small, delicate systems for language models.

- Prefer declarative structure over sprawling code.
- Request the smallest command, workflow, or patch that moves the task forward.
- Keep context light. Read and modify only the surfaces required for the job.
- Treat `/cmd` as durable reusable ability and `/workflows` as task-specific programs.
- Favor commands that become more capable through parameters instead of multiplying narrow commands.
- Make proposals easy to inspect, easy to edit, and easy to discard.
- When a structure becomes too bulky to author comfortably, introduce or improve a builder instead of dumping more raw markup.
- Be concrete about paths, parameters, inputs, outputs, and verification.
